# LocalDts 数据预览与界面布局优化建议

## 1. 引言

LocalDts 作为一款数据迁移工具，其核心价值在于高效、准确地完成数据迁移任务。然而，在数据迁移过程中，用户往往需要对数据进行预览、验证和配置，以确保迁移的正确性。因此，提供直观、友好的数据预览功能和优化的界面布局对于提升用户体验至关重要。本建议文档将结合前期的项目评估和优化建议，重点阐述数据预览功能的实现思路以及针对数据展示需求的界面布局优化方案。

## 2. 数据预览功能实现建议

数据预览功能应允许用户在实际迁移之前，查看从源系统提取的数据样本，以及经过转换器处理后的数据形态。这有助于用户验证配置的正确性，并及时发现潜在的数据问题。

### 2.1 预览数据流的接入点

建议在以下关键环节提供数据预览入口：

- **数据源配置后**：用户配置完数据源（`IDataSource`）后，可预览原始数据样本。
- **每个转换器配置后**：在应用每个 `ITransformer` 后，用户可预览数据经过该转换器处理后的效果。
- **目标源配置前**：在数据即将写入目标源（`IDataTarget`）前，预览最终的数据形态。

### 2.2 预览数据获取机制

为了避免在预览时加载全部数据造成性能问题，建议采用以下策略：

- **限制预览数据量**：只提取前 N 条（例如 50-100 条）数据进行预览。这可以通过在 `IDataSource` 的 `ExtractAsync` 方法中增加一个可选参数 `previewLimit` 来实现，或者在 `MigrationEngine` 中对 `IAsyncEnumerable` 进行 `Take(N)` 操作。
- **异步加载**：确保数据预览操作不会阻塞 UI 线程，利用 `async/await` 模式。

### 2.3 预览数据展示组件

WPF 提供了强大的数据绑定和 UI 组件，适合展示表格数据：

- **DataGrid**：这是 WPF 中最常用的表格数据展示控件，支持列的自动生成、排序、过滤和自定义模板。对于数据预览，`DataGrid` 是一个理想的选择。
  - **优点**：功能丰富，易于绑定 `IEnumerable<DataRecord>`。
  - **优化建议**：结合 **HandyControl** 或 **MaterialDesignInXamlToolkit** 等第三方库，可以显著提升 `DataGrid` 的外观和交互体验，例如提供更好的分页、加载动画和上下文菜单。

## 3. 界面布局优化建议（针对数据展示）

当前的 WPF 界面（`MainWindow.xaml`）采用了侧边导航栏和主内容区域的经典布局。在数据展示方面，可以进一步优化布局，以提供更高效、更美观的用户体验。

### 3.1 整体布局策略

- **Master-Detail 布局**：对于配置和预览场景，可以采用 Master-Detail 布局。例如，左侧列表展示可用的插件或配置项（Master），右侧区域展示对应插件的详细配置界面或数据预览（Detail）。
- **选项卡式布局**：在单个页面内，如果需要展示不同维度的数据或配置，可以使用选项卡（TabControl）进行组织，例如“原始数据预览”、“转换后数据预览”、“错误数据”等。

### 3.2 关键页面布局优化

#### 3.2.1 插件管理页面 (`PluginManagerPage`)

- **列表展示**：使用 `DataGrid` 或 `ListView` 展示已加载的插件列表，包含插件名称、ID、版本和类型等信息。
- **详细信息/操作区域**：在列表下方或右侧，提供一个区域显示选中插件的详细信息，并提供加载/卸载插件、查看插件文档等操作按钮。

#### 3.2.2 数据源/目标源/转换器配置页面

- **配置表单**：使用清晰的表单布局，利用 `Grid` 和 `StackPanel` 组织配置项。对于复杂的配置项，可以考虑使用可折叠的面板（`Expander`）。
- **实时验证**：对配置项进行实时输入验证，并提供友好的错误提示。
- **预览区域**：在配置区域下方或右侧，集成一个数据预览区域，当用户完成关键配置后，可以点击“预览”按钮查看数据样本。

#### 3.2.3 任务执行页面 (`TaskExecutionPage`)

- **进度展示**：使用进度条（`ProgressBar`）和文本标签实时显示迁移进度，包括总记录数、已处理记录数、错误记录数等。
- **日志输出**：提供一个可滚动的文本区域（`TextBox` 或 `RichTextBox`）显示详细的迁移日志，方便用户追踪问题。
- **错误/警告列表**：如果发生错误或警告，以列表形式（`ListView` 或 `DataGrid`）展示，包含错误类型、发生时间、受影响的记录等信息，并允许用户导出。

### 3.3 界面美化与用户体验提升

结合前期的建议，引入先进的开源 WPF 插件库是提升 UI/UX 的关键。

- **HandyControl** \[1]：
  - **控件丰富**：提供大量美观且功能强大的控件，如 `DataGrid`、`TextBox`、`Button`、`ComboBox`、`Dialog` 等的现代化样式。
  - **主题支持**：支持亮色/暗色主题切换，提升视觉舒适度。
  - **动画效果**：内置多种动画效果，使界面交互更加流畅。
- **MaterialDesignInXamlToolkit** \[2]：
  - **Material Design 风格**：遵循 Google Material Design 指南，提供一致且现代的视觉风格。
  - **图标支持**：集成 Material Design Icons，方便在界面中使用丰富的矢量图标。
  - **易于集成**：通过简单的资源引用即可应用到现有项目。

**实施步骤**：

1. **安装 NuGet 包**：在 `DataMigration.Wpf` 项目中安装 `HandyControl` 或 `MaterialDesignThemes` NuGet 包。
2. **引入资源字典**：在 `App.xaml` 中引入所需的主题和控件样式资源字典。
3. **替换默认控件**：逐步将默认的 WPF 控件替换为库提供的样式或控件，例如将 `Button` 替换为 `hc:Button` 或 `materialDesign:Button`。
4. **自定义样式**：根据项目需求，对引入的控件进行进一步的样式定制。

## 4. 总结

通过在关键环节引入数据预览功能，并结合现代 WPF UI 库对界面布局进行优化，LocalDts 将能提供更直观、更高效的用户体验。这不仅能帮助用户更好地控制数据迁移过程，也能显著提升工具的专业性和吸引力。建议在开发过程中，持续关注用户反馈，迭代优化界面设计。
